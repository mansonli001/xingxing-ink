### v1.0 产品契约

**完整闭环**：
1. 三档人格对话（随便聊 / 讲道理 / 扇巴掌）+ 12 问框架
2. 聊够 6 轮 + 覆盖 4 题 → 出诊断书（「值得晒的 BP」门槛）
3. 不达标 → 弹窗内嵌追问表单 → 答完一键出 BP
4. 诊断书 = PART 1/2/3 结构化报告 + 长图分享

**技术契约**：
- API 路径稳定不破坏
- DiagnosisReport / QLedger / StatsSummary schema 稳定
- mind repo 四层注入架构稳定

**品牌契约**：
- 不哄人，只怼人
- 三档人格，三种姐姐视角
- 永不空洞 BP，永不水诊断书

## v0.8.0 · PRD 模式（下一个版本 · 已完成产品设计）

### 一句话定位

> **「和醒醒聊一聊你的 idea，¥0.99 换一份能直接丢给 AI 开发的 PRD。」**

> 注：MVP 验证阶段**先不收费**，全功能开放跑用户行为数据。

### 核心设计要点（本次对话已对齐）

1. **付费机制**：MVP 阶段不收费，全功能开放；远期价格锚 ¥0.99 / 单次对话（cookie 模式 = 本次对话解锁，不登录）
2. **付费入口**：参考 [toxic-pm.bijin.ink](http://toxic-pm.bijin.ink/) 的「Pro 角标 + 生成 PRD 按钮」明牌设计，**姐姐永远不主动引导付费**
3. **拼图引擎**：12 格通用（对齐 v6.0 SOP-03 定性 4 项 + SOP-02 竞品 3 项 + 模板 A 功能 5 项），**对所有用户都开**
4. **PRD 模板**：模板 A · 12 项 AI 开发说明书版（V1 只做这一个）
5. **PRD 形态**：**完整文章**（不是纪要 / 不是 bullet list） · 网页可编辑 + PDF 下载 + 一键复制 prompt 给 AI
6. **「已聊透」横幅**：拼图 ≥ 10/12 触发显示
7. **生成 PRD 按钮**：默认 disabled，拼图够了变绿激活
8. **「不管了先上线」彩蛋按钮**：拼图未满也能强制出 PRD（半成品标注，留逃生口）
9. **拼图未满时点付费**：软拒绝——"醒醒还没挤干你的 idea，回去多聊几轮，按钮等你回来"
10. **PRD 标题**：正式标题 + 页脚"本 PRD 由醒醒陪你骂出来的 · YYYY-MM-DD"

### 12 格拼图清单

```
需求定性层（对齐 SOP-03）
① 目标用户（能指认的具体画像）
② 使用场景（3 个含时间+动作+痛点）
③ 盈利模式（三选一）
④ 产品边界（P0 做 + 二期不做）

竞品差异层（对齐 SOP-02）
⑤ 竞品优点
⑥ 竞品缺点
⑦ 差异化

说明书层（对齐模板 A）
⑧ 用户现有痛点（3 条）
⑨ 核心价值（2-3 条）
⑩ MVP 功能（≤5，带"输入→处理→输出"）
⑪ 页面结构（≤3）
⑫ UI 风格（四选一）

默认固定：技术栈 Next.js + DeepSeek + Vercel
```

### 关键约束

- 醒醒 prompt 一字不改（v0.7.0 反套路化已稳定）
- 弹药库本就把"PRD"作为攻击武器之一（"这是 ChatGPT 帮你写的 PRD 吧"），**不禁言**
- 拼图状态用 LocalStorage（cookie 匿名，零成本）
- 拼图标签器异步并行调 DeepSeek，不阻塞主对话流

### 工作流咬合（重要战略意义）

```
v6.0 副业 SOP 工作流
SOP-01 选题 → SOP-02 竞品 → SOP-03 定性 → SOP-3.5 验证 → SOP-04 PRD → SOP-05 AI 开发 → SOP-06 上线
                              └──────── ◆ 醒醒产品在这里接管 ◆ ────────┘
                                       （SOP-03 + SOP-04 由用户和醒醒聊完成）
```

醒醒 v0.8.0 PRD 模式 = **mansonli001 副业工具链的入口产品**，第一个"用自己产品做出来"的产品，自循环。

## 决策记录（重要节点的判断）

### 2026-05-10 · v0.7.0 反套路化决策

**用户提出痛点**：「我担心模板固定后，不像真人聊天，第二次就觉得套路一致。」

**最终方案**（不重做架构，纯 prompt 层 + 极轻量代码层）：
- 三档 prompt 末尾追加 4 条铁律：跟着用户走 / 招牌不连续 / 第3轮意外动作 / 轮次种子
- chat stream route 注入"今日切入点"伪随机系统提示（10 个切入点池，每 2 轮换一个，sessionId 哈希驱动）

**风险评估**：低风险——所有改动都是 append-only，未触及现有 415/489/531 行三档 prompt 正文。

### 2026-05-10 · v0.8.0 PRD 模式延后决策

**最初方向偏移**：曾一度想"把 PRD 模式塞进 v2.0 一锅端"，被用户纠正。

**最终决策**：v0.8.0 单独做 PRD 模式，v2.0 留作综合大版本。理由：
- "2.0 是大版本，按版本顺序更新，没到那么远"
- 小步快跑节奏稳，每个版本只做一件事
- v0.7.0 反套路化先跑稳，再上 PRD 才有底

### 2026-05-10 · 付费机制延后到 v2.0

**MVP 验证阶段策略**：v0.8.0 全功能不收费，跑用户行为数据；v2.0 大版本才接付费。
理由：在还不知道 PRD 出来的质量稳不稳定时，**¥0.99 也是承诺**——宁可先验证再收费。
